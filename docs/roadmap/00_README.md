# 发表路线图 (Publication Roadmap) — 索引

本目录每个文件对应**一个可能发论文的方法**，只回答一件事：
**这个方法要做哪些事、解决哪些问题，才能达到发表标准。**

分工（避免和其它文档重复）：
- 分数、逐 epoch 曲线 → `EXPERIMENT_SUMMARY.md`（唯一数据源）
- 方法机制说明 → `docs/methods/*.md`
- 全局下一步/项目级诊断 → `docs/NEXT_STEPS_*.md`
- **本目录**：单个方法的"发论文待办 + 缺陷 + 缺失实验 + 可解释性 + 发表门槛"

每个文件统一 8 节：
1. 定位（一句话 + 代码 + 注册名）
2. 当前状态（分数是否可信、是否已验证）
3. 方法缺陷 / 已知问题（诚实记录，审稿人会问的）
4. 缺失实验清单（发论文必需）
5. 可解释性怎么做到位
6. 发表门槛 checklist
7. 目标分区（条件性）+ 撞车/竞品文献
8. 一句话结论：现在离发表还差什么

> ⚠️ 通用前提（对所有方法都成立）：
> - "几区"是期刊属性，不是 idea 属性；本目录所有分区都是**条件预测**。
> - BLCA 单癌种 + C-index ~0.65 目前**只够 2 区**；上 1 区必须有**多癌种一致增益 + 战胜公开 SOTA（MOTCat/SurvPath/OTSurv 同口径）+ 多 seed 复现**。
> - 2026-07 修复了两个影响全体的 bug：OT eps 调度间断、batch=4 早峰（梯度累积/warmup/裁剪）。**修复前的崩溃分数一律作废**，需用 `scripts/run_fix_verify_fold02.sh` 重跑后才算数。
