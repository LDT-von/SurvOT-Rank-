+### 14.4 NaN 修复 (`535e020`)
+
+fold 2 在 score-first 配方下每 8 epoch 必定 NaN 崩溃。`535e020` 修复内容：
+
+- DCT log-Sinkhorn 对 NaN/Inf/极端 cost 做有限值投影
+- 防止极端 cost 污染整个 batch
+- 无可比较事件对时返回中性指标
+- 评估时按完整患者行过滤非有限预测
+
+修复后 fold 2 成功跑完 50 epoch，且 fold 2 不再是从 epoch 0 开始滑坡（best@18），证明 NaN 不仅是崩溃问题，也毒化了前几个 epoch 的梯度。
+
+### 14.5 核心结论
+
+1. **简化就是力量**：7-loss → 2-loss (NLL + IPCW rank)，best +0.04, last5 +0.05, gap -24%。
+2. **OT 在 score-first 配方下仍是活跃的正则化项** (ot ≈ 1.1, 非零)，但不像 v3.2 那样作为主损失参与竞争。
+3. **anchor 机制是过拟合加速器**：v3.2 去掉 anchor 后 gap 从 0.095 → 0.062；v3.3 彻底移除后 gap 进一步降到 0.073。
+4. **fold 3/4 末期回到峰值** (0.705/0.757) 是 v3.3 区别于 v3.2 的最大质变 — 模型不再"过拟合完立即崩塌"。
+5. **fold 间差异大**：best 0.705~0.770, gap 0.012~0.173 — BLCA n=380 很小，fold 1 的 0.716 比 fold 4 的 0.770 低 0.05 是正常的小样本噪声。
